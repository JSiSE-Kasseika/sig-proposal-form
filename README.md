# 教育システム情報学会 第二種SIG提案書 入力フォーム

教育システム情報学会の第二種SIG提案書をWeb上で入力できるReactアプリケーションです。

## 主な機能

- **フォーム入力**: 提案書の全項目をWeb上で入力可能
- **動的データ管理**:
  - 提案者（幹事・メンバー）の追加・削除
  - 学術的問いの追加・削除
  - 年間活動スケジュールの追加・削除
- **企画セッション管理**:
  - 複数の大会・研究会から実施予定を選択（チェックボックス）
  - 選択したセッションの詳細情報入力
- **文字数カウント**: 概要や効果などの長文フィールドで文字数を表示
- **データ管理**:
  - 自動保存（ローカルストレージ）
  - JSONエクスポート/インポート
  - データクリア機能

## セットアップ

### 必要な環境
- Node.js (v14以上推奨)
- npm または yarn

### インストール

```bash
# 依存パッケージのインストール
npm install

# または
yarn install
```

### 開発サーバーの起動

```bash
npm start

# または
yarn start
```

ブラウザで http://localhost:3000 が自動的に開きます。

### ビルド

```bash
npm run build

# または
yarn run build
```

本番用にビルドされたファイルが `build/` ディレクトリに生成されます。

## 使い方

1. **入力**: フォームの各項目を入力します
   - 入力内容は自動的にブラウザのローカルストレージに保存されます

2. **提案者の管理**:
   - 「+ 提案者を追加」ボタンで新しい提案者を追加
   - 役割（筆頭幹事/幹事/メンバー）をドロップダウンから選択
   - 「削除」ボタンで不要な提案者を削除

3. **企画セッションの選択**:
   - 実施予定の大会・研究会にチェック
   - チェックした場合、詳細情報の入力欄が表示されます

4. **データのエクスポート**:
   - 「JSONでエクスポート」ボタンでデータをJSONファイルとしてダウンロード
   - バックアップや他の環境への移行に利用できます

5. **データのインポート**:
   - 「JSONからインポート」ボタンで以前エクスポートしたJSONファイルを読み込み

6. **データのクリア**:
   - 「データをクリア」ボタンですべての入力内容をリセット

## データ構造

エクスポートされるJSONファイルには以下の情報が含まれます：

- 提案日
- SIG名称（和・英・略称）
- 提案者リスト
- 研究テーマの概要
- 学術的問いのリスト
- 期待される効果
- 初回企画セッション情報
- 企画セッション実施予定（チェックボックス選択）
- 企画セッション詳細
- 年間活動スケジュール

## 技術スタック

- React 18
- CSS3 (カスタムスタイル)
- ローカルストレージAPI

## 注意事項

- データはブラウザのローカルストレージに保存されます
- ブラウザのキャッシュをクリアするとデータが失われる可能性があります
- 定期的にJSONエクスポート機能でバックアップを取ることをおすすめします
